System Overview

  • Hybrid encryption combines AES-256-CBC for data encryption and RSA-2048 for key exchange
  • Each transaction uses a unique AES session key encrypted with the bank's RSA public key
  • All communication is encrypted end-to-end over untrusted networks
  • Bank decrypts the session key using RSA private key, then decrypts transaction data

RSA-2048 Key Pair

  • Padding: OAEP with SHA-256

  • Never share the private key publicly
  • Public key can be distributed to all ATMs

About This System

End-to-End Encryption for ATM-Bank Communication Over Untrusted Networks

Project Overview

Purpose

This system implements a secure communication protocol for ATM-Bank transactions using hybrid encryption. It ensures that sensitive transaction data remains confidential even when transmitted over untrusted networks.

Technical Architecture

Encryption Process

  • Step 1: Generate random AES-256 session key
  • Step 2: Encrypt transaction data with AES-256-CBC
  • Step 3: Encrypt AES session key with RSA-2048 (bank's public key)
  • Step 4: Transmit encrypted package (data + key + IV)

Decryption Process

  • Step 1: Receive encrypted package
  • Step 2: Decrypt AES session key using RSA private key
  • Step 3: Decrypt transaction data using AES-256-CBC
  • Step 4: Process decrypted transaction
